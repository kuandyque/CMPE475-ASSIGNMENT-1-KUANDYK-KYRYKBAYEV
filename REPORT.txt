1. INTRODUCTION
====================================================================

This report presents a comparative analysis of eight machine learning algorithms applied to a TV shows and movies dataset. The objective is to classify content as either a "Movie" or a "TV Show" using extracted features from the dataset, evaluate each algorithm using five classification metrics, and visualize the results through plots.

The dataset used is tv-shows.csv, a collection of 9,338 titles from streaming platforms including Netflix. The dataset is in tabular CSV format and falls under the Tabular modality category.


====================================================================
2. DATABASE SELECTION
====================================================================

2.1 Dataset Description

The dataset contains 9,338 rows and 13 columns. Each row represents a single media title (movie or TV show) with the following attributes:

- id: Unique identifier for each title
- type: Content type - "Movie" or "TV Show" (this is our classification target)
- title: Name of the content
- director: Director(s) of the content (30.6% missing)
- cast: Actors in the content (10.0% missing)
- country: Country of production (10.3% missing)
- date_added: Date the title was added to the platform
- release_year: Year the content was originally released
- rating: Content rating (PG-13, TV-MA, R, etc.)
- duration: Duration in minutes (for movies) or number of seasons (for TV shows)
- listed_in: Genre categories
- description: Brief description of the content
- platform: Streaming platform name

2.2 Why This Dataset?

This dataset was chosen because:
(a) It is in tabular CSV format, which is compatible with all eight required algorithms.
(b) It contains a natural binary target variable (Movie vs TV Show).
(c) It has sufficient size (9,338 rows) for meaningful training and evaluation.
(d) It contains diverse feature types (numeric, categorical, text) that require practical feature engineering.
(e) It contains real-world imperfections like missing values, requiring data cleaning.

2.3 Class Distribution

The target variable (type) has the following distribution:
- Movie: 6,528 samples (69.9%)
- TV Show: 2,810 samples (30.1%)

This moderate class imbalance (approximately 2.3:1) justifies the use of multiple evaluation metrics beyond simple accuracy.


====================================================================
3. TARGET SELECTION
====================================================================

3.1 Classification vs Regression

Classification was chosen over regression for the following reasons:
(a) The target variable (type) is categorical with exactly two classes.
(b) Six of the eight required algorithms (Naive Bayes, Decision Trees, Random Forest, Logistic Regression, KNN, SVM) are primarily designed for classification.
(c) Classification metrics (Accuracy, Precision, Recall, F1-Score, ROC-AUC) provide more nuanced performance analysis than regression metrics.
(d) Classification results are more intuitive to interpret and visualize (confusion matrices, ROC curves).

3.2 Target Variable

The type column serves as the target variable:
- Class 0: Movie
- Class 1: TV Show

The model's task is to predict whether a given title is a Movie or a TV Show based on its metadata features.


====================================================================
4. FEATURE ENGINEERING
====================================================================

Machine learning algorithms require numeric input. The raw dataset contains text, dates, and categorical values that must be transformed into meaningful numeric features. Thirteen features were engineered:

4.1 release_year (Direct Use)
The release year is already numeric and used directly. It captures temporal trends in content production.

4.2 duration_num (Extracted from duration)
The numeric value is extracted from the duration string using regular expression. For example, "128 min" becomes 128 and "3 Seasons" becomes 3. This is the most discriminative feature because movies have values in the range 60-300 (minutes) while TV shows have values in the range 1-15 (seasons).

4.3 rating_encoded (Label-Encoded)
Content ratings (TV-MA, PG-13, R, TV-Y, etc.) are converted to integers using scikit-learn's LabelEncoder. Movies tend to use MPAA ratings while TV shows use TV-specific ratings.

4.4 country_US (Binary)
A binary flag indicating whether "United States" appears in the country field. The US has different production ratios of movies to TV shows compared to other countries.

4.5 country_India (Binary)
A binary flag indicating whether "India" appears in the country field. India produces a disproportionately high number of movies (Bollywood).

4.6 num_cast (Count)
The number of cast members listed, calculated by counting commas in the cast string and adding 1. TV shows often list larger ensemble casts.

4.7 has_director (Binary)
A binary flag indicating whether a director is listed. TV shows are far more likely to have no director listed (since they have multiple directors across seasons). Of the 2,855 missing director values, a disproportionate number belong to TV shows.

4.8 num_genres (Count)
The number of genre tags listed in the listed_in field. TV shows tend to be tagged with more categories.

4.9 genre_drama (Binary)
A binary flag for drama-related content, detected by searching for "Drama" in the listed_in field.

4.10 genre_comedy (Binary)
A binary flag for comedy-related content, detected by searching for "Comed" in the listed_in field.

4.11 genre_international (Binary)
A binary flag for international content, detected by searching for "International" in the listed_in field.

4.12 month_added (Extracted from date_added)
The month component of the date_added field (1-12), extracted using pandas datetime parsing. Captures seasonal content release patterns.

4.13 description_len (Count)
The character count of the description text. Different content types may have different description length conventions.


====================================================================
5. DATA PREPROCESSING
====================================================================

5.1 Missing Value Handling

Missing values were handled as follows:
- Numeric features: Filled with 0 using fillna(0)
- Text features: Filled with empty string before feature extraction
- Target variable: No missing values (0 missing)

5.2 Train/Test Split

The dataset was split 80/20:
- Training set: 7,470 samples (80%)
- Test set: 1,868 samples (20%)

Stratified sampling (stratify=y) was used to maintain the same 70/30 class ratio in both sets. A fixed random seed (random_state=42) ensures reproducibility.

5.3 Feature Scaling

StandardScaler was applied to normalize all features to zero mean and unit variance. This is critical for distance-based algorithms (KNN, SVM) where features with larger ranges would dominate distance calculations. The scaler was fit on training data only and then applied to both training and test data to prevent data leakage.


====================================================================
6. EVALUATION METRICS
====================================================================

Five classification metrics were used to evaluate each algorithm:

6.1 Accuracy
Formula: (TP + TN) / Total
Measures the overall percentage of correct predictions. Simple and intuitive but can be misleading on imbalanced datasets.

6.2 Precision
Formula: TP / (TP + FP)
Measures the proportion of positive predictions that are correct. High precision means few false positives (false alarms).

6.3 Recall (Sensitivity)
Formula: TP / (TP + FN)
Measures the proportion of actual positives that are correctly identified. High recall means few missed cases.

6.4 F1-Score
Formula: 2 * (Precision * Recall) / (Precision + Recall)
The harmonic mean of Precision and Recall. Provides a single score that balances both metrics. More informative than accuracy on imbalanced datasets.

6.5 ROC-AUC (Area Under the ROC Curve)
Measures the model's ability to distinguish between classes across all possible decision thresholds. AUC = 1.0 indicates perfect classification; AUC = 0.5 indicates random guessing.


====================================================================
7. ALGORITHMS
====================================================================

7.1 Bayesian (Naive Bayes) - GaussianNB

Naive Bayes is a probabilistic classifier based on Bayes' Theorem. It calculates the posterior probability of each class given the observed features, assuming feature independence. The Gaussian variant assumes features follow normal distributions. Despite the "naive" independence assumption, it performs well on many real-world tasks and is extremely fast to train.

7.2 Decision Trees - DecisionTreeClassifier(max_depth=5)

Decision Trees build a flowchart-like structure where each internal node tests a feature, each branch represents a decision, and each leaf assigns a class. The algorithm selects the feature and threshold at each node that best separates the classes using metrics like Gini impurity. The max_depth=5 parameter limits tree depth to prevent overfitting. Decision Trees are highly interpretable but can be unstable.

7.3 Random Forest - RandomForestClassifier(n_estimators=100)

Random Forest is an ensemble method that builds 100 independent Decision Trees, each trained on a random subset of the data and features. Predictions are made by majority voting across all trees. This approach reduces overfitting and variance compared to a single Decision Tree, producing more robust and accurate predictions.

7.4 Simple Linear Regression - LinearRegression() with 1 feature

Simple Linear Regression fits a straight line y = mx + b using a single feature (release_year). Since it outputs continuous values rather than class labels, a threshold of 0.5 is applied for classification. "Simple" means exactly one independent variable is used. This baseline demonstrates the limitation of using a single feature for classification.

7.5 Multiple Linear Regression - LinearRegression() with all features

Multiple Linear Regression extends Simple Linear Regression to use all 13 features: y = w1*x1 + w2*x2 + ... + w13*x13 + b. Each feature receives its own weight. The output is again thresholded at 0.5. This demonstrates how adding more features improves predictive power, even with a simple linear model.

7.6 Logistic Regression - LogisticRegression(max_iter=1000)

Despite its name, Logistic Regression is a classification algorithm. It applies a sigmoid function to the linear combination of features, producing a probability between 0 and 1. If the probability exceeds 0.5, the sample is classified as TV Show. Logistic Regression is the proper classification counterpart to Linear Regression. The max_iter=1000 parameter allows sufficient iterations for convergence.

7.7 KNN (K-Nearest Neighbors) - KNeighborsClassifier(n_neighbors=5)

KNN classifies a new sample by finding its 5 nearest neighbors in feature space (using Euclidean distance) and assigning the majority class among those neighbors. KNN is a "lazy learner" that stores all training data and performs computation at prediction time. It requires scaled features because distance calculations are sensitive to feature magnitudes.

7.8 SVM (Support Vector Machine) - SVC(kernel='rbf', probability=True)

SVM finds the optimal hyperplane that maximally separates the two classes. It focuses on "support vectors" (training points closest to the boundary). The RBF (Radial Basis Function) kernel allows SVM to handle non-linear boundaries by mapping data into a higher-dimensional space. The probability=True parameter enables probability estimates needed for ROC-AUC calculation.

8.2 Analysis

Best performing algorithms:
- Decision Trees and Random Forest tied for highest accuracy (0.9968) and F1-Score (0.9947).
- Random Forest achieved perfect recall (1.0000), correctly identifying every TV Show in the test set.
- Random Forest, Logistic Regression, and SVM all achieved ROC-AUC above 0.9998.

Worst performing algorithm:
- Simple Linear Regression performed poorly (Accuracy = 0.6991) because using only release_year as a single feature provides insufficient information. It predicted "Movie" for all samples, resulting in zero Precision, Recall, and F1 for the TV Show class.

Key insight:
- The dramatic improvement from Simple to Multiple Linear Regression (0.6991 to 0.9625 accuracy) demonstrates that using multiple features provides significantly more predictive power.

Why are scores so high?
- The duration_num feature provides near-perfect class separation. Movies are measured in minutes (typically 60-300) while TV shows are measured in seasons (typically 1-15). Most algorithms can easily exploit this strong signal.

10. CONCLUSION
====================================================================

This assignment demonstrated the application of eight machine learning algorithms to a binary classification task using streaming platform data. Key findings include:

1. Random Forest and Decision Trees achieved the highest overall performance, with Random Forest achieving perfect recall.
2. Ensemble methods (Random Forest) outperformed individual algorithms in terms of stability and ROC-AUC.
3. Simple Linear Regression's poor performance highlights the importance of using multiple features.
4. Logistic Regression, as the proper classification counterpart to Linear Regression, achieved near-perfect results.
5. Feature engineering transforming raw text and categorical data into numeric features was essential for algorithm compatibility.
6. The duration_num feature was the most discriminative, providing near-perfect class separation.
7. Feature scaling was critical for distance-based algorithms (KNN, SVM) but unnecessary for tree-based methods.
